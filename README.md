## ğŸ“¦ Deployment

### GitHub Pages

1. **Update `docusaurus.config.ts`**:
   - Ganti `url` dengan URL GitHub Pages Anda (contoh: `https://your-username.github.io`)
   - Ganti `baseUrl` dengan nama repo jika diperlukan (contoh: `/kisi-kisi-belajar/`)
   - Ganti `organizationName` dengan username GitHub Anda
   - Ganti `projectName` dengan nama repo GitHub Anda

2. **Install gh-pages** (opsional, untuk deployment otomatis):
   ```bash
   npm install --save-dev gh-pages
   ```

3. **Deploy**:
   ```bash
   npm run deploy
   ```

   Atau secara manual:
   ```bash
   GIT_USER=<GITHUB_USERNAME> npm run deploy
   ```

### Netlify

1. Push code ke GitHub
2. Login ke [Netlify](https://www.netlify.com/)
3. Pilih "New site from Git"
4. Pilih repository Anda
5. Build settings:
   - Build command: `npm run build`
   - Publish directory: `build`
6. Deploy!

### Vercel

1. Push code ke GitHub
2. Login ke [Vercel](https://vercel.com/)
3. Import project dari GitHub
4. Vercel akan otomatis detect Docusaurus
5. Deploy!

## ğŸ”§ Konfigurasi

### Mengubah Informasi Site

Edit `docusaurus.config.ts`:
- `title`: Judul website
- `tagline`: Tagline website
- `url`: URL production
- `baseUrl`: Base URL (untuk GitHub Pages biasanya `/repo-name/`)

### Menambahkan Konten

1. Tambahkan file `.md` di folder `docs/`
2. Update `sidebars.ts` untuk menambahkan ke sidebar
3. Atau gunakan auto-generated sidebar dengan `autogenerated`
